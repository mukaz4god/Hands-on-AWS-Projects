# Day 38 – AWS Artifact Compliance

# Interview Ready Wording

## Question: What is AWS Artifact?

**Answer:**  
AWS Artifact is the AWS portal for accessing AWS compliance reports and agreements, such as SOC reports, ISO certifications, and PCI-related documentation.

---

## Question: Does AWS compliance mean my workload is automatically compliant?

**Answer:**  
No. AWS compliance helps with the security and compliance of the cloud, but customers are responsible for how they configure and operate their workloads in the cloud.

---

## Question: What is the shared responsibility model?

**Answer:**  
The shared responsibility model defines which security and compliance responsibilities belong to AWS and which belong to the customer. AWS secures the cloud infrastructure, while the customer secures their data, identities, configurations, applications, and workloads.

---

## Question: How would you use AWS Artifact in an enterprise environment?

**Answer:**  
I would use AWS Artifact to retrieve official AWS compliance evidence for audits, customer assurance, risk assessments, and governance reviews, while securely handling downloaded reports.
