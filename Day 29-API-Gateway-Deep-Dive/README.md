# Day 37 – API Gateway Deep Dive

## Project Goal

To build a secure REST API architecture with Lambda integration, throttling, API keys, usage controls, and resource policies.

## Services Used

- Amazon API Gateway
- AWS Lambda
- CloudWatch Logs
- IAM
- Usage Plans
- API Keys

## Outcome

A production-style API Gateway deployment was implemented, providing secure API access, controlled traffic consumption, and policy-based restrictions for serverless backends.

## Key Learning

- API Gateway can securely expose Lambda.
- Throttling protects APIs from abuse.
- Usage plans control client consumption.
- Resource policies restrict who can invoke the API.
