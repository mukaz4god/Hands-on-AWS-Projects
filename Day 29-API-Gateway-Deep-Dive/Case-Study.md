# API Gateway Deep Dive

## Case Study – Building a Secure Serverless API Architecture

### Business Scenario

A company exposes backend functionality to partners using an API. The backend runs on AWS Lambda, but the API must be protected from abuse and unauthorized access.

The business wants API access controlled through API keys, usage plans, throttling, and resource policies.

### Security Requirements

- Create a REST API.
- Integrate API Gateway with Lambda.
- Configure throttling and rate limits.
- Require API keys.
- Apply resource policies.
- Test secure API access.

### Your Role

You are the Cloud Engineer responsible for building and securing the API Gateway architecture.

### Project Goal

Build a secure API architecture that exposes Lambda through API Gateway while controlling access, rate limits, and usage.
