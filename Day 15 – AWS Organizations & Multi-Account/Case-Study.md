# Business Scenario

KFJ is a growing enterprise that manages multiple AWS workloads (Dev, Test, Prod). Managing everything in a single account creates risks:
- Lack of isolation
- Poor governance
- Billing complexity
- Security control gaps

The organisation wants a multi-account strategy with centralised governance.

# Task

You are the Cloud Security Engineer implementing:

- AWS Organization
- Multiple accounts (Dev, Prod)
- Service Control Policies (SCPs)
- Consolidated billing
