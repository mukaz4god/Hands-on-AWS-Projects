# AWS Artifact Compliance

## Case Study – Enterprise Compliance Awareness

### Business Scenario

A company is preparing for an enterprise customer security review. The customer asks for evidence of AWS compliance posture, including SOC reports, ISO certifications, PCI-related reports, and a clear explanation of the shared responsibility model.

The security team needs to know where to access AWS compliance documents and how to map AWS responsibilities against customer responsibilities.

### Compliance Requirements

- Explore AWS Artifact.
- Locate SOC reports.
- Locate ISO certifications.
- Locate PCI compliance reports.
- Map the shared responsibility model.
- Document what AWS covers and what the customer must configure.

### Your Role

You are the Cloud Security / Governance Engineer responsible for preparing cloud compliance evidence and explaining AWS shared responsibility.

### Project Goal

Develop enterprise compliance awareness by using AWS Artifact and mapping compliance documents to real AWS customer responsibilities.
