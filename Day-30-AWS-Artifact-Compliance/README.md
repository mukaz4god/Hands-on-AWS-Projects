# Day 38 – AWS Artifact Compliance

## Project Goal

To develop enterprise compliance awareness by exploring AWS Artifact reports, reviewing AWS assurance documentation, and mapping the shared responsibility model.

## Services Used

- AWS Artifact
- AWS Compliance Reports
- Shared Responsibility Model

## Outcome

A compliance-awareness workflow was established, enabling structured review of AWS audit reports and a clearer understanding of AWS and customer control responsibilities.

## Key Learning

- AWS Artifact provides compliance evidence.
- AWS compliance does not automatically make a workload compliant.
- Customers must configure security controls correctly.
- Shared responsibility must be understood for audits and enterprise reviews.
